# DID Landscape Explorer — Codex

## What This Is

Vue 3 + Vite + Tailwind 4 app. Community-sourced DID method visibility tool — shows what method communities have documented. Not a registry, not a grading tool.

## Critical Rules

1. **"Not Assessed" ≠ "Fails"** — `assessed: false` → show gray "Not Assessed", NEVER a grade.
2. **No scoring in components** — all logic in `composables/` and `config/`. Components only render.
3. **5-state assessment model** — `met`, `partial`, `not-met`, `not-applicable`, `not-assessed`.
4. **No opinions** — show observable facts only. No computed maturity levels, no rankings.
5. **MethodologyNote visible on every view** — users must always see the disclaimer.
6. **Taxonomy is provisional** — subject to W3C DID WG review.

## Styling Rules (STRICT)

All visual styling MUST be defined as semantic CSS classes in `style.css`. Components MUST NOT use raw Tailwind color/size utilities directly.

### Text Hierarchy (defined in style.css)
- `.text-primary` — white, headings, method names
- `.text-secondary` — gray-200, body text, table cells
- `.text-description` — gray-300, descriptions, secondary content
- `.text-label` — gray-400, 10px uppercase tracking-wider, section labels
- `.text-muted` — gray-400, help text, footnotes
- `.text-dim` — gray-500, decorative only (row numbers, disabled states)
- `.text-mono` — monospace, for method names and requirement IDs

### Component Classes (defined in style.css)
- `.card` — rounded-lg border border-gray-700 bg-gray-800/30
- `.card-stat` — stat card with centered value + label
- `.badge-lifecycle` — rounded-full border, color from config
- `.badge-category` — rounded-full border border-gray-600 text-gray-300
- `.badge-tier-core/common/specialized` — requirement tier pills
- `.badge-industry` — colored border + text per industry
- `.pill-supported/unsupported/unknown` — feature/crypto status pills
- `.btn-primary/success/outline/ghost/accent/disabled` — button variants
- `.input-base` — bg-gray-800 border border-gray-700 rounded-lg
- `.callout-info/warning/success` — bordered callout boxes
- `.modal` — overlay + centered card pattern

### Color Semantics
- Green = met/supported/active
- Yellow = partial/deprecated
- Red = not-met/unsupported
- Blue = info/links/primary actions
- Purple = experimental
- Gray = no data/not assessed

### Contrast Rules (bg-gray-950)
- Readable: white, gray-100 through gray-400
- Decorative only: gray-500 (row numbers, disabled)
- NEVER: gray-600+ for content text

## Data

`methods-index.json` → `MethodEntry` interface in `types/method.ts`:
- `assessed: boolean` — has the method community submitted data?
- `requirements: Record<string, RequirementAssessment>` — per-requirement status + evidence
- `maturity.lifecycle` — community-declared (NOT computed)
- `feat` — feature booleans (null = no data, not false)
- `ucScores` — only displayed when `assessed: true`
